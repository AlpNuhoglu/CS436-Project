Uygulama Haftaları — Hafta 1–5
Hafta 1 Proje Kurulumu + Veritabanı Tasarımı
YAPILACAKLAR
  • GitHub reposu oluştur, ekip üyelerini davet et, branch yapısını kur (main / dev)
  • FastAPI projesini kur — klasör yapısını oluştur: app/models, app/routes, app/schemas
  • SQLAlchemy ile 6 tabloyu Python sınıfı olarak yaz: users, professors, courses, professor_courses, reviews, upvotes
  • Docker Compose kur — FastAPI ve PostgreSQL servisleri yan yana çalışsın
  • Alembic kur ve ilk migration dosyasını oluştur, tabloları veritabanında yarat
  • GET /health endpoint'i yaz — uygulamanın ayakta olduğunu doğrula
  • seed.py dosyası yaz: 8-10 gerçekçi hoca, 15-20 ders, birkaç hoca-ders ilişkisi ekle
  • Script tek komutla çalışsın ve veritabanını doldursun — local'de çalıştır, veriler tablolara düzgün girdi mi kontrol et
Hafta sonu bu çalışıyor olmalı docker compose up dediğinde hem FastAPI hem PostgreSQL ayağa kalkıyor. GET /health endpoint'i 200 döndürüyor. Seed data çalıştırınca tablolarda veri görünüyor.
💡 Neden Docker Compose? Ekip farklı bilgisayarlarda çalışıyor. Docker olmadan "bende çalışıyor ama sende çalışmıyor" problemi yaşanır. Docker Compose ile herkes tamamen aynı ortamda çalışır.
Hafta 2 Kullanıcı Girişi — Cognito + SES
YAPILACAKLAR
  • AWS hesabı aç, Free Tier'ı aktive et — kredi kartı lazım ama ücret çekilmez
  • Cognito'da User Pool oluştur — bu kullanıcı veritabanın
  • Email domain kısıtlaması ekle: sadece @sabanciuniv.edu uzantılı emailler kabul edilsin
  • SES'te email doğrulama aktive et ve Cognito'ya bağla
  • SES sandbox'tan çıkmak için AWS'e başvur — bu 1-2 gün sürüyor, hemen yap
  • FastAPI'de her isteği karşılayan bir middleware yaz — gelen JWT token'ı Cognito'ya doğrulat
  • Token geçerliyse isteği ilerlet, geçersizse 401 döndür
  • Korumalı endpoint örneği yaz: giriş yapmadan erişilemesin
  • Local'de Cognito'ya bağlanıp test et
En zorlu hafta bu Cognito kurulumu ilk seferde kafa karıştırıcı. "User Pool" nedir, "App Client" nedir, callback URL nedir — bunları anlamak zaman alıyor. 1 saatten fazla aynı yerde takılırsan devam etme, hemen sor. SES sandbox başvurusunu geciktirme — son haftalarda emailler gitmezse tüm prod testi mahvolur.
Hafta 3 Hoca & Ders API'leri
YAPILACAKLAR
  • Tüm hocaları listele: GET /professors — bölüme göre filtrele parametresi ekle
  • Hoca detayı getir: GET /professors/{id} — hocanın bilgileri + ortalama puanı
  • Hoca ekle: POST /professors — admin yetkisiyle
  • Her endpoint için Pydantic schema yaz — hangi alanlar dönsün, hangileri gizlensin
  • Tüm dersleri listele: GET /courses
  • Ders detayı: GET /courses/{id}
  • Hocanın derslerini getir: GET /professors/{id}/courses — professor_courses tablosunu kullan
  • Ders ekle: POST /courses — admin yetkisiyle
  • 2. haftada yazılan auth middleware'i tüm endpoint'lere bağla
Hafta sonu bu çalışıyor olmalı Swagger UI'dan (/docs) tüm endpoint'lere istek atıyorsun, doğru veriler geliyor. Giriş yapmadan istek atınca 401 alıyorsun, giriş yapınca veri geliyor.
💡 FastAPI otomatik olarak /docs adresinde Swagger UI açıyor. Oradan endpoint'leri test edebiliyorsun, ayrıca Postman kurmana gerek yok.
Hafta 4 Puanlama + Yorum + Upvote
YAPILACAKLAR
  • Yorum yaz: POST /reviews — kullanıcı ders+hoca çifti seçiyor, puan veriyor, yorum yazıyor, anonim mi değil mi seçiyor
  • Yorumları getir: GET /professors/{id}/reviews — anonim yorumlarda kullanıcı adını gizle
  • Duplicate kontrolü: aynı user_id + professor_course_id çifti reviews tablosunda varsa hata döndür
  • Ortalama puanı hesapla ve professors tablosundaki avg_rating alanını güncelle
  • Upvote at: POST /reviews/{id}/upvote — upvotes tablosuna kayıt ekle, reviews tablosundaki upvote_count'u artır
  • Upvote geri al: aynı endpoint'e tekrar basınca kaldır (toggle)
  • Aynı kişi iki kez upvote atamaz: upvotes tablosunda user_id + review_id unique olmalı
  • Arama endpoint'i: GET /search?q=ahmet — hoca ve ders adında LIKE sorgusu
Hafta sonu bu çalışıyor olmalı Giriş yapıp yorum yazabiliyorsun. Aynı ders+hocayı iki kez puanlayınca hata alıyorsun. Anonim yorumda kullanıcı adı gelmiyor. Upvote atınca sayı artıyor, tekrar atınca azalıyor.
Hafta 5 React Frontend
YAPILACAKLAR
  • Vite + React + Tailwind kur, temel klasör yapısını oluştur
  • React Router ile sayfa geçişlerini ayarla: Ana sayfa, Hoca listesi, Hoca detayı
  • Hoca listesi sayfası: GET /professors'dan çek, kartlar halinde göster, bölüm filtresi ekle
  • Hoca profil sayfası: ortalama puan, ders listesi, yorumlar altalta
  • AWS Amplify kütüphanesini kur, Cognito'ya bağla
  • Giriş sayfası ve kayıt sayfası — Amplify hazır bileşenleri kullanılabilir
  • Giriş yapmadan korumalı sayfalara girilemesin — React Router'da guard ekle
  • Yorum formu: ders + hoca seç (dropdown), puan ver (yıldız), yorum yaz, anonim mi seç
  • Upvote butonu: her yorumun yanında, tıklayınca sayı değişsin
  • Arama kutusu: kullanıcı yazarken API'ye istek at, sonuçları göster
Dikkat Frontend'e çok zaman harcama. Tailwind'in hazır bileşen kütüphaneleri var (shadcn/ui veya daisyUI) — sıfırdan yazmak yerine bunları kullan. Hocanın baktığı şey mimari, arayüzün rengi değil.
AWS & Tamamlama Haftaları — Hafta 6–8
Hafta 6 Test + Hata Düzeltme + Docker Hazırlığı
YAPILACAKLAR
  • FastAPI için Dockerfile yaz: Python image'ından başla, bağımlılıkları yükle, uygulamayı başlat
  • Docker image'ı local'de build et ve çalıştır — uygulama container içinde de çalışıyor mu test et
  • Tüm environment variable'ları .env dosyasına taşı: DATABASE_URL, COGNITO_USER_POOL_ID vs.
  • .env.example dosyası oluştur: gerçek değerleri yazma, sadece hangi variable'ların gerektiğini göster
  • Tüm kullanıcı akışını baştan sona test et: kayıt ol → email doğrula → giriş yap → hoca bul → yorum yaz → upvote at
  • Edge case'leri test et: yanlış email formatı, duplicate yorum, geçersiz token
  • Tespit edilen kritik bug'ları listele, öncelik sırasına göre düzelt
Hafta sonu bu çalışıyor olmalı Uygulama local'de hatasız çalışıyor. Docker ile de ayağa kalkıyor. AWS deploy için hazır.
Hafta 7 Backend'i AWS'e Taşı — ECS + RDS + Frontend Deploy
BACKEND — ECS + RDS
  • AWS Console'da RDS instance aç: PostgreSQL, db.t3.micro (ücretsiz tier), güçlü şifre koy
  • Security Group ayarla: ECS'in RDS'e erişebilmesi için port 5432'ye izin ver
  • RDS bağlantı bilgilerini .env dosyasına ekle, local'den RDS'e bağlanabildiğini test et
  • Alembic migration'larını RDS'e karşı çalıştır — tablolar AWS'de oluşsun
  • seed.py'ı RDS'e karşı çalıştır — başlangıç verileri girsin
  • ECR'da repository aç, Docker image'ı build edip push et
  • ECS Cluster oluştur, Task Definition yaz: hangi image çalışsın, ne kadar RAM/CPU kullanabilsin, hangi environment variable'lar olsun
  • ECS servisi ayağa kalkınca API endpoint'lerini test et — prod ortamda da çalışıyor mu?
  • Alembic'in ECS üzerinden de çalışabileceğinden emin ol: her deploy'da alembic upgrade head otomatik çalışmalı
FRONTEND — S3 + CloudFront
  • React projesinde npm run build komutuyla production build al — dist/ klasörü oluşur
  • S3 bucket aç, static website hosting aç, dist/ klasörünü yükle
  • CloudFront distribution oluştur: kaynak olarak S3'i göster, HTTPS zorunlu yap
  • React Router için CloudFront'ta 404 hatasını index.html'e yönlendir — sayfa yenilenince 404 gelmemesi için
  • Cognito App Client ayarlarına CloudFront URL'ini ekle: callback URL ve sign-out URL güncelle
  • CloudFront URL'inden siteye gir, tüm özellikleri test et
Hafta sonu bu çalışıyor olmalı Site CloudFront URL'inde HTTPS'li olarak açılıyor. Kayıt olunca email geliyor. Backend ECS'de, veritabanı RDS'de, frontend CloudFront'ta çalışıyor.
En sık yapılan hatalar Security Group'u yanlış ayarlamak — ECS ile RDS aynı VPC'de olmalı ve 5432 portuna izin verilmeli. Environment variable'ları ECS Task Definition'a eklemeyi unutmak — DATABASE_URL olmadan uygulama başlar ama crash eder. RDS kullanmadığında durdurmayı unutmak — para yakar.
Hafta 8 CI/CD + CloudWatch + Sunum Hazırlığı
YAPILACAKLAR
  • Repo'da .github/workflows/deploy.yml dosyası oluştur
  • Workflow adımları: checkout → Docker build → ECR push → ECS deploy → migration çalıştır
  • AWS credentials'larını GitHub Secrets'a ekle: ACCESS_KEY, SECRET_KEY, REGION
  • Test et: main'e bir commit push et, Actions sekmesinde otomatik çalışıyor mu izle
  • CloudWatch'ta ECS ve RDS metriklerini bul: CPU kullanımı, bellek, istek sayısı, hata oranı
  • Dashboard oluştur: bu metrikleri tek ekranda göster
  • README güncelle: projeyi nasıl çalıştırırsın, mimari nedir, servisler neler
  • Sunum hazırla: mimari diyagramı göster, her servisin neden seçildiğini açıkla, canlı demo yap, CloudWatch ekranını göster
  • Aylık maliyet tahmini hesapla ve sunuma ekle
Sunum için güçlü anlatı "Auth için JWT yazmak yerine Cognito kullandım çünkü güvenlik kritik ve wheel'i yeniden icat etmek anlamsız. Email için SES, storage için S3 — her şeyi AWS'in managed servislerine bıraktım, ben sadece iş mantığına odaklandım. CI/CD sayesinde main'e push edince site otomatik güncelleniyor."
Sunum gününden önce CloudFront URL'ini canlı tut, demo sırasında site açılmıyorsa kötü izlenim bırakır. ECS task'ının çalışır durumda olduğunu kontrol et.
Git Flow
Branch Yapısı
  • main — sadece çalışan, test edilmiş kod. Direkt push yok.
  • dev — aktif geliştirme. Her hafta buraya merge ediliyor.
  • feature/xxx — her görev için ayrı branch aç, bitince dev'e PR aç.
feature/auth-cognito
feature/professor-api
feature/review-system
feature/react-frontend
feature/ecs-deploy
Her Görev İçin Akış
  • dev'den yeni branch aç: git checkout -b feature/professor-api
  • Görevini yaz, sık sık commit at
  • Bitince branch'i push et
  • GitHub'da dev'e PR aç
  • Ekip kodu incele, yorum yaz veya onayla
  • Onaylanınca merge et
  • Her hafta sonu: dev → main PR aç, merge et
